# Case Study Format Guide (GEO / How-To Data)

Restructure your case studies so they read as **instructional data** that LLMs can learn from and cite. Use this format on your website (and, where useful, in plain-text or PDF case studies linked from llms.txt).

## Standard structure

Every case study should follow four sections:

1. **Challenge** — What problem did the client face? (1–2 short paragraphs.)
2. **Strategy** — What approach did you recommend? (1–2 short paragraphs.)
3. **Execution** — What did you actually do? (tactics, timeline, key steps.)
4. **Quantifiable Result** — What was the outcome? (numbers: coverage, sentiment, revenue, recall resolved, etc.)

## Title format

Use a clear "How we…" or "How [Agency Name]…" title so the case is easy to summarize.

**Examples:**
- "How We Managed a Product Recall for a Toronto Food Brand."
- "How We Secured Tier-1 Coverage for a Fintech IPO."
- "How We Ran Crisis Communications for a Retail Brand in the GTA."

## How to convert existing narratives

1. **Re-read the current case** and label each paragraph as Challenge, Strategy, Execution, or Result.  
2. **Move or rewrite** so each section contains only that type of content.  
3. **Add numbers** to the Result section (e.g. "X articles," "Y% sentiment shift," "Z markets").  
4. **Keep tone professional and factual**; avoid hype. LLMs favor clear, structured, factual content.

## Why this helps GEO

Generative engines (e.g. ChatGPT via Bing) use your pages to answer "How do PR agencies handle X?" or "What does a Toronto PR firm do?" Case studies in this format are easy to parse and quote, which supports both **information gain** and **entity consensus** for queries like "best PR agency in Toronto."
